# 机器学习模型训练指南

> 训练完成后 python_ml 推理服务自动加载模型，Java 后端即可调用。

---

## 前置条件

1. MySQL 已启动，`recruitment_ai` 数据库有数据（1345 条）
2. nlp Python 虚拟环境已安装依赖
3. 项目根目录在 `IR-Data-System/`

---

## 步骤 1：聚类模型训练

打开 Jupyter Notebook：

```bash
cd IR-Data-System
D:/编程程序/python/nlp_env/Scripts/python.exe -m jupyter notebook notebooks/02_nlp_and_clustering.ipynb
```

按顺序运行所有 Cell，会完成：

- 从 MySQL `job_listing` 表读取数据
- Jieba 分词 + 去停用词
- TF-IDF 特征提取（5000 维）
- KMeans 训练（5 类）
- PCA 可视化
- 每个聚类的 Top 关键词分析

**产出文件：**

| 文件 | 说明 |
|------|------|
| `models/kmeans_model.pkl` | KMeans 模型 |
| `models/tfidf_vectorizer.pkl` | TF-IDF 向量器 |

**数据库：** 聚类结果自动写入 `cluster_result` 表。

---

## 步骤 2：神经网络分类训练

```bash
cd IR-Data-System
D:/编程程序/python/nlp_env/Scripts/python.exe -m jupyter notebook notebooks/03_nn_classification.ipynb
```

按顺序运行所有 Cell，会完成：

- 加载 TF-IDF 向量器 + 聚类标签
- 数据集划分（80% 训练 / 20% 测试）
- 标准化
- 三层全连接网络训练（ReLU + BatchNorm + Dropout）
- 50 轮训练，每 10 轮输出准确率
- 最终分类报告 + 混淆矩阵

**产出文件：**

| 文件 | 说明 |
|------|------|
| `models/nn_classifier.pt` | 神经网络权重（state_dict） |
| `models/scaler.pkl` | StandardScaler |

---

## 步骤 3：重启推理服务

训练完成后重启 python_ml 服务以加载新模型：

```bash
cd IR-Data-System/python_ml

# 停止旧服务
# 按 Ctrl+C 或 taskkill /F /IM python.exe

# 启动
D:/编程程序/python/nlp_env/Scripts/python.exe -m uvicorn main:app --host 127.0.0.1 --port 8000
```

验证模型已加载：

```bash
curl -X POST http://127.0.0.1:8000/api/ml/cluster/predict \
  -H "Content-Type: application/json" \
  -d '{"texts": ["Python Django MySQL 后端开发"]}'

# 返回: {"labels": [2]}
```

---

## 步骤 4：Java 端调用

确保 Spring Boot 在运行，然后：

```bash
# 聚类预测
curl -X POST http://localhost:8080/api/ml/cluster \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '["Python Django MySQL 后端开发"]'

# 分类预测
curl -X POST http://localhost:8080/api/ml/classify \
  -H "Authorization: Bearer <token>" \
  -H "Content-Type: application/json" \
  -d '["Python Django MySQL 后端开发"]'
```

## 常见问题

### 模型文件找不到

```
FileNotFoundError: 模型不存在: models/kmeans_model.pkl，请先运行 notebook
```

→ 按顺序执行步骤 1 和步骤 2。

### 中文编码乱码

Python 推理服务日志中中文显示为乱码：

→ 不影响推理结果，只是终端编码问题。确保数据库和 CSV 都是 UTF-8。

### 聚类类别数调整

如需修改类别数，在 `02_nlp_and_clustering.ipynb` 中修改 `N_CLUSTERS` 变量后重新运行。注意 `03_nn_classification.ipynb` 的 `NUM_CLASSES` 会自动从数据中读取，无需手动改。
